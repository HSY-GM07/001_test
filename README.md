# ⚔️ 중세 판타지: 태그 매치 RPG 콘솔 게임

객체지향 프로그래밍(OOP) 원칙과 다양한 C# 컬렉션을 활용하여 구현한 포켓몬스터 스타일의 턴제 콘솔 RPG 게임입니다. 사용자는 3인의 캐릭터 파티를 전략적으로 교체하며 던전의 몬스터들을 격파해야 합니다.

---

## 주요 구현 목표
* **OOP 핵심 원칙 적용**: 상속, 추상화, 다형성을 활용한 확장성 높은 구조 설계
* **포켓몬식 턴제 UI**: 상위 메뉴(싸운다, 아이템, 교대)와 하위 메뉴(기술 선택)의 유기적 흐름 구현
* **전략적 시스템**: 직업 간 2배 상성 데미지 및 딕셔너리 기반 소모품 인벤토리 연동

---

## 파일 및 폴더 구조
* **`Program.cs`** : 게임의 오프닝, 스테이지 전환 및 전반적인 시나리오 제어
* **`Interfaces/`**
  * `IUsable.cs` : 아이템 확장성을 위한 규격 인터페이스
* **`Models/`**
  * `Character.cs` (Abstract) : 플레이어 캐릭터의 베이스 클래스 및 상성 연산 로직 보유
  * `Warrior.cs` / `Mage.cs` / `Rogue.cs` : 직업별 스탯 및 고유 기술 3종 구현
  * `Monster.cs` : 적 몬스터의 스탯 및 공격 메커니즘 구현
* **`Systems/`**
  * `BattleManager.cs` : 전투 화면 출력(`Console.Clear`), 행동 선택, 턴 제어 및 캐릭터 강제 교체 처리

---

## ⚙️ 핵심 핵심 기능

### 1. 상성 시스템 (2배 배율)
* 전사(`Warrior`) ➔ 도적(`Rogue`) ➔ 마법사(`Mage`) ➔ 전사(`Warrior`) 구조로 물고 물리는 상성 관계 정립
* 상성 우위 시 기본 데미지의 **200%** 적용

### 2. 포켓몬 스타일 전투 UI & 뒤로가기
* 메인 전투 프레임에서 `싸운다`, `아이템`, `교대`, `게임 종료` 선택 가능
* 기술 메뉴 진입 후 `뒤로가기` 입력 시, 턴 소모 없이 메인 메뉴로 복귀하는 루프 연산 구현

### 3. 인벤토리 및 드롭 시스템
* `Dictionary<string, int>`를 활용해 소모품(체력 포션)의 수량을 실시간 제어
* 스테이지 클리어 시 캐릭터 휴식(HP 회복)과 함께 아이템 추가 드롭 연동

## 🛠️ 문제 해결 및 회고
* **출력 가속 문제**: 콘솔창에 텍스트가 순식간에 지나가는 현상을 `Console.Clear()`와 `Console.ReadLine()`의 적절한 배치로 해결하여 턴제 게임의 호흡을 확보했습니다.
* **이메일 프라이버시 제한(Git Push 에러)**: GitHub 푸시 과정에서 `GH007` 보안 에러를 마주했으나, 로컬 Git의 `user.email` 설정을 GitHub 전용 노리플라이(Noreply) 대체 주소로 변경하고 커밋 저자를 재설정(`--amend`)하여 정상적으로 원격 저장소에 반영했습니다.
* **향후 계획**: `IUsable` 인터페이스를 확장한 다양한 장비 아이템 추가, 기술별 상태 이상(스턴, 독) 디버프 로직 고도화를 진행할 계획입니다.
